# Logbook Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a personal Logbook section as the new homepage — a reverse-chron feed of short notes and long-form posts, with optional EN/IT bilingual toggle per entry.

**Architecture:** A Jekyll collection (`_logbook`) provides the entries. The existing `index.html` is replaced with a logbook feed. The news feed moves to `/news/` (no pagination — GitHub Pages' `jekyll-paginate` v1 only works at `/`). A single-entry layout handles bilingual content by splitting on an HTML comment marker.

**Tech Stack:** Jekyll (Minima theme), Liquid templates, vanilla JS (language toggle only), GitHub Pages

---

## Important context before you start

- Site lives in `docs/` — that is the Jekyll root. All Jekyll files are under `docs/`.
- Global CSS: `docs/_sass/minima/custom-styles.scss`
- Base layout: `docs/_layouts/base.html` — wraps all pages with header, ASCII brand, footer
- Header nav: `docs/_includes/header.html`
- Dark theme: black `#000000` bg, amber `#ffb000` brand, `JetBrains Mono` font, muted white `#999` body text
- Existing collections in `_config.yml`: `devlogs` (output: false)
- `jekyll-paginate` v1 limitation: paginator only works on the root `index.html`. Since logbook replaces `/`, news at `/news/` will use a plain Liquid loop instead.
- Bilingual split marker: `<!-- lang-split -->` — an HTML comment in the Markdown body. Kramdown passes it through to rendered HTML unchanged. The layout splits `content` on this string.

## Task 1: Register `_logbook` collection + remove unused paginate config

**Files:**
- Modify: `docs/_config.yml`

**Step 1: Add logbook collection and remove paginate**

In `docs/_config.yml`, find the `collections:` block and add logbook:

```yaml
collections:
  logbook:
    output: true
    permalink: /logbook/:year/:month/:day/:title/
  devlogs:
    output: false
```

Also remove these two lines (they only worked when news was at `/` and we no longer need them):
```yaml
paginate: 15
paginate_path: "/page/:num/"
```

And remove `jekyll-paginate` from the plugins list:
```yaml
plugins:
  - jekyll-feed
  - jekyll-seo-tag
  - jekyll-sitemap
```

**Step 2: Verify config is valid YAML**

Run: `cd docs && bundle exec jekyll build 2>&1 | head -20`
Expected: No YAML parse errors. Build may warn about missing files — that's fine for now.

**Step 3: Commit**

```

```

## Task 2: Create `logbook-entry.html` layout — single entry with bilingual toggle

**Files:**
- Create: `docs/_layouts/logbook-entry.html`

**Step 1: Create the layout**

```html
---
layout: base
---

<article class="logbook-entry-full">

  {% if page.title and page.title != "" %}
  <h1 class="logbook-entry-title">{{ page.title }}</h1>
  {% endif %}

  <div class="logbook-entry-meta">
    <span class="logbook-entry-date">{{ page.date | date: "%Y-%m-%d" }}</span>
    {% if page.category %}
    <span class="logbook-cat">[{{ page.category }}]</span>
    {% endif %}
  </div>

  {% if page.languages %}
    {% assign parts = content | split: "<!-- lang-split -->" %}
    {% assign content_en = parts[0] %}
    {% assign content_it = parts[1] %}

    <div class="lang-toggle">
      <button class="lang-btn lang-btn--active" onclick="setLang('en', this)" aria-pressed="true">EN</button>
      <button class="lang-btn" onclick="setLang('it', this)" aria-pressed="false">IT</button>
    </div>

    <div class="lang-block lang-block--en lang-block--visible">{{ content_en }}</div>
    <div class="lang-block lang-block--it">{{ content_it }}</div>

    <script>
      function setLang(lang, btn) {
        document.querySelectorAll('.lang-btn').forEach(function(b) {
          b.classList.remove('lang-btn--active');
          b.setAttribute('aria-pressed', 'false');
        });
        btn.classList.add('lang-btn--active');
        btn.setAttribute('aria-pressed', 'true');
        document.querySelectorAll('.lang-block').forEach(function(b) {
          b.classList.remove('lang-block--visible');
        });
        document.querySelector('.lang-block--' + lang).classList.add('lang-block--visible');
      }
    </script>

  {% else %}
    <div class="logbook-entry-body">{{ content }}</div>
  {% endif %}

</article>
```

  {% else %}

      {% if entry.excerpt %}

## Known limitations

- **No pagination on `/news/`** — GitHub Pages `jekyll-paginate` v1 only works at the root. News shows the 30 most recent posts via `| limit: 30`. Acceptable for now.
- **No pagination on logbook feed** — logbook shows all entries. Fine until there are many.
- **`excerpt` in logbook feed** — Jekyll auto-generates excerpt from the first paragraph. For bilingual entries this will always be the English excerpt (it comes first). Acceptable.
